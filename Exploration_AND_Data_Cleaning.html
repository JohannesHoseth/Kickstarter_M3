Choosing the features¶

So what should we keep? We've inspected the dataset, containing all of the above columns. First, we found various columns that contains almost exclusively NaNs, which we deleted. Additionally, we wanted to make predictions on the Kickstarter success, with no information that is not known when launching a campaign. This implied that all columns pertaining information about the final amount of money that have been pledged were deleted.

Left were the following columns, used as features for the various models.

  • ID - An identification code.
  • Name - The (string) name of the project.
  • blurb - The (string) description of the project.
  • launched_at, created_at, deadline - three columns with date data.
  • disable_communication - A (boolean) choice by creators, opting to NOT communicate with their backers.
  • is_starable - A (boolean) choice by Kickstarter, allowing certain campaigns to achieve a 'starred' recommendation.
  • static_usd_rate - A rate pertaining currency info specifically to USD (the currency used for all purposes here).
  • fx_rate - the foreign exchange rate.
  • goal - the monetary goal of the campaign to become successful.
  • category - A dictionary containing main and sub categories.
  • country - (string) country name
  • photo - photo URLs, to be scraped for Computer vision algorithm
  • staff pick - A (boolean) recommendation by Kickstarter
  • Target: state (sucessful/failed) - the very thing we try to predict

The plotting utilized the 'density=True' parameter to normalize the two histograms. This gives us the relative frequency distribution, as opposed to a mere absolute frequency distribution, in order to truly compare the 'succesful' 'failed' datasets that quite different value counts.

As we can see, a lot more succesful campaigns in the first half of the graph, whereas for the bins indicating the largest goals, there are more failed campaigns.
